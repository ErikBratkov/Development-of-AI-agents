# Домашнее задание к уроку №11 "Графовая память и сложный контекст // ДЗ"

Память агента

## Цель:

Реализовать поиск по данным и извлечение связей для повышения точности ответа;

## Описание/Пошаговая инструкция выполнения домашнего задания:

- Подготовьте небольшой набор данных;
- Реализуйте векторный поиск;
- Добавьте хранение связей между объектами;
- Реализуйте гибридный поиск (по смыслу + по связям);
- Проверьте работу на примере запроса;

## Формат сдачи:

- код + пример запроса/ответа

## Критерии оценки:

Задание считается принятым, если:

- работает поиск по данным;
- реализованы связи между объектами;
- показан результат ответа;

## Компетенции:
Организовывать управление контекстом и памятью агента

- реализовывать поиск и хранение знаний.
- сохранять отношения между объектами и строить гибридный поиск через векторную и графовую память.

---

## Решение

Решение выполнено как доработка демо-сервиса голосового ассистента,
код лежит в папке `voice-assistant-v2`, план доработки -
в `ARCHITECTURE.md` рядом с этим файлом.

Что сделано по шагам задания:

- набор данных - за основу взят подкинутый генератор
  `voice-assistant-v2/data-generator` (пакет `graph_generator`):
  синтетический доменный граф (50 сотрудников, 6 компаний,
  12 технологий, 15 проектов) и документы с фрагментами, seed
  фиксирован. Скрипт `seed_knowledge.py` добавляет к нему эмбеддинги
  и векторный индекс и грузит все в Neo4j
- векторный поиск - фрагменты документов эмбеддятся локальной моделью
  sentence-transformers (`app/embeddings.py`) и ищутся через
  нативный векторный индекс Neo4j (`app/knowledge.py`)
- хранение связей - доменный граф в Neo4j с отношениями WORKS_AT,
  KNOWS, WORKS_ON, OWNED_BY, USES, DEPENDS_ON, фрагменты документов
  связаны с сущностями через MENTIONS
- гибридный поиск - инструмент `search_knowledge`: сначала фрагменты
  по смыслу вопроса, затем факты обходом графа от упомянутых
  сущностей, результат уходит LLM через function calling
- проверка на примере - вопрос "Кто из сотрудников знает Neo4j и в
  каких компаниях они работают?", пример запроса и ответа описан в
  `voice-assistant-v2/README.md`, раздел "База знаний в
  Neo4j". Принадлежность людей к компаниям есть только в связях
  графа, поэтому чисто векторный поиск такой вопрос не закрывает

Запуск: `docker compose up -d`, затем `python seed_knowledge.py` и
`python main.py` из папки сервиса (подробности в README).
