# Генератор графовых данных

Генерирует тестовый граф и выгружает его в Neo4j. Это подкинутый к
уроку проект, он взят за основу наполнения базы знаний сервиса:
скрипт `../seed_knowledge.py` использует пакет `graph_generator`
напрямую, добавляя к нему эмбеддинги фрагментов и векторный индекс.
Граф состоит из двух слоев:

- доменный: Company, Person, Project, Technology и бизнес-связи
  (WORKS_AT, KNOWS, USES, OWNED_BY, WORKS_ON, DEPENDS_ON)
- лексический: Document и Chunk со структурными связями
  (PART_OF, NEXT_CHUNK, FIRST_CHUNK)

Слои сшиты связью MENTIONS от фрагмента текста к доменной сущности.
Документы собираются шаблонами из уже сгенерированных фактов, поэтому
упоминания известны точно и извлекать их через NER не нужно.

Важная особенность текстов: связи WORKS_AT, WORKS_ON, OWNED_BY и
USES в них сознательно не проговариваются, эти факты живут только
в графе. Так на вопросах вида "кто знает X и где работает" виден
выигрыш гибридного поиска перед чисто векторным.

## Структура

- `graph_generator/config.py` - описание предметной области:
  справочники и параметры генерации
- `graph_generator/models.py` - доменная модель и графовое представление
- `graph_generator/generator.py` - класс GraphGenerator, возвращает
  объектную модель без привязки к графу, включая документы
- `graph_generator/graph.py` - адаптер модели в узлы и связи
- `graph_generator/exporter.py` - выгрузка в Neo4j, без APOC
- `main.py` - точка входа

## Локальный Neo4j

Сервер поднимается через docker-compose из корня сервиса
(файл `voice-assistant-v2/docker-compose.yml`, уровнем выше):

```
docker compose up -d
```

Креды прописаны в docker-compose.yml и годятся только для демо:

- логин: `neo4j`
- пароль: `graphdemo42`
- bolt: `bolt://localhost:7687`
- браузер: http://localhost:7474

APOC не требуется, используется обычный официальный образ.
Данные живут в volume `neo4j_data` и переживают перезапуск,
`docker compose down -v` удаляет их вместе с volume.

## Запуск

Зависимости: faker, neo4j. Обе входят в группу rag сервиса,
из корня сервиса их ставит `pip install -e ".[rag]"`.

Обычный сценарий - запуск `python seed_knowledge.py` из корня
сервиса, он грузит этот же датасет вместе с эмбеддингами. Автономный
запуск без эмбеддингов:

```
export NEO4J_URI=bolt://localhost:7687
export NEO4J_USER=neo4j
export NEO4J_PASSWORD=graphdemo42

python main.py
```

Без NEO4J_URI скрипт только печатает статистику по данным.
Выгрузка идемпотентна (MERGE по id), повторный запуск безопасен.

## Тесты

Тесты не требуют Neo4j, работают с объектной моделью напрямую.
