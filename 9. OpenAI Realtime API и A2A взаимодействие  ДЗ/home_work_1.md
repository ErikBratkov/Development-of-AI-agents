# Домашнее задание к уроку №9 "OpenAI Realtime API и A2A взаимодействие // ДЗ"

Подключение контекста и streaming

## Цель:

Реализовать передачу контекста и потоковый ответ агента;

## Описание/Пошаговая инструкция выполнения домашнего задания:

- Создайте простой источник контекста (файл или API);
- Подключите его к агенту;
- Реализуйте streaming ответа;
- Добавьте простой сценарий взаимодействия;
- Зафиксируйте архитектуру подключения;

## Формат сдачи:

- код + короткое видео/скрин выполнения

## Критерии оценки:

Задание считается принятым, если:

- реализована передача контекста;
- работает streaming ответа;
- сценарий воспроизводим;

## Компетенции:

Обеспечивать интеграцию и коммуникацию агентных систем

- подключать инструменты и контекст через протокол
- реализовывать асинхронное и потоковое взаимодействие

---

## Решение

Решение - демо-сервис голосового ассистента в папке `voice-assistant`.
Монолит на FastAPI: браузер шлет голос по WebSocket, сервер распознает
его локально (faster-whisper), отправляет текст в LLM через OpenRouter
и стримит обратно токены ответа и озвучку (Piper). Кроме голоса есть
текстовый ввод, он работает без микрофона и аудио-зависимостей.

Как закрыты критерии приемки:

- источник контекста (API) - у модели есть инструмент `get_weather`
  (`voice-assistant/app/weather.py`), через function calling она сама
  ходит в Open-Meteo API (геокодинг города плюс текущая погода) и
  отвечает по полученным данным, цикл вызова инструментов живет в
  `voice-assistant/app/llm_client.py`
- передача контекста - контекст диалога (system prompt, rolling
  summary, последние реплики) собирается в `voice-assistant/app/memory.py`
  и передается в модель на каждом ходе, при разрастании история
  сжимается в summary
- streaming ответа - токены LLM и аудио TTS стримятся в браузер по
  одному WebSocket по мере генерации
- сценарий взаимодействия - пошаговый сценарий демонстрации описан в
  `voice-assistant/README.md` (раздел "Сценарий для демонстрации"),
  там же инструкции по установке и запуску под Linux и Windows
- архитектура подключения зафиксирована в `ARCHITECTURE.md` рядом с
  этим файлом (транспорт, потоки данных, схема модулей)

Проверка сценария: сказать или написать "Привет, меня зовут Эрик",
дождаться ответа, затем спросить "Как меня зовут?" - ассистент помнит
контекст. Вопрос "Какая погода в Питере?" покажет вызов внешнего API
через инструмент.
