# Домашнее задание к уроку №3 "Бенчмарки и data-driven роутинг от теории к коду ДЗ"

Проектирование архитектуры агента

## Цель:

спроектировать архитектуру ИИ-агента с учетом цикла выполнения и выбора моделей для прикладной задачи;

## Описание/Пошаговая инструкция выполнения домашнего задания:

Для выполнения домашнего задания выберите один из кейсов или предложите свой.

Определите прикладной сценарий агента (например: обработка запросов, анализ данных, генерация кода);
Опишите цикл работы агента по схеме Reason → Act → Observe;
Выделите этапы, где требуется использование разных моделей;
Опишите простую логику роутинга (например: простые задачи → дешевая модель, сложные → продвинутая);
Зафиксируйте архитектуру в виде схемы или структурированного описания;

## Формат сдачи:

ссылка на документ (схема + краткое описание)
Примерное время выполнения
2–3 часа

Код не обязателен.
Продумайте шаги агента, разбейте задачу на подзадачи, продумайте как защититься от Dead Loop,
какие модели вы возьмёте, почему, какие лимиты у вашего провайдера LLM, в какой момент Агент свалится,
как защититься, попробуйте посчитать экономику (хотя бы плюс минус, посмотрите сколько стоит 1млн токенов,
посмотрите размер вашего промпта, что будет в системном промпте? документы, инструкции, кодовая база, законы, примеры и
прочее)

## Критерии оценки:

Задание считается принятым, если:

- описан цикл работы агента;
- выделена логика выбора моделей;
- архитектура соответствует прикладному сценарию;

## Компетенции:

Проектировать архитектуру ИИ-агентов под прикладные задачи

- формировать цикл работы агента для решения задач.
- выбирать и проектировать схему развертывания и роутинга моделей.
